# Rc&lt;T&gt;：引用计数

> 《Rust 程序设计语言》：[`Rc<T>`：引用计数智能指针](https://doc.rust-lang.org/book/ch15-04-rc.html)

## 使用 `Rc<T>` 共享单线程所有权

- 引用计数(reference counting，rc)，通过记录一个数据被引用的次数来确定该数据是否正在被使用

- 当引用次数归零时，就代表该数据不再被使用，因此可以被清理释放

- 希望在堆上分配一个对象供程序的多个部分使用且无法确定哪个部分最后一个结束时，就可以使用 Rc 成为数据值的所有者

- 当希望在堆上分配一个对象，并让程序的多个部分共享它的所有权时，可以使用`Rc<T>`

- `Rc<T>` 通过引用计数让每个使用方都成为数据值的所有者，从而避免悬垂引用

- `Rc::clone` 仅仅复制了智能指针并增加了引用计数，并没有克隆底层数据

下面的示例模拟一个资源缓存：

- `ResourceCache` 使用 `HashMap<String, Rc<Resource>>` 保存已加载资源
- 预览功能和导出功能都从缓存中取得同一个资源
- 缓存可以先移除资源；只要预览或导出仍然持有 `Rc<Resource>`，资源就不会被释放
- 当最后一个 `Rc<Resource>` 离开作用域时，资源才会被清理

## `Rc<T>` 使用限制

- 不支持多线程共享
- 不直接允许修改内部值
